# Cursed Pit — Design Roadmap

**Статус:** Draft v0.9
**Назначение:** центральный навигатор по дизайну. Список всех систем, их статус, приоритет, зависимости.
**Применимость:** для использования между сессиями обсуждения как точка отсчёта.

---

## 1. Как читать этот документ

Каждая система имеет:
- **Статус** — насколько проработана
- **Приоритет** — насколько срочно нужно для текущего этапа разработки
- **Режим** — для какого режима актуальна
- **Зависимости** — что должно быть готово до того, как обсуждать эту
- **Ключевые открытые вопросы** — что нужно решить

### Условные обозначения статуса

- ✅ **Зафиксировано** — есть design doc, обсуждено и принято
- 🔶 **Частично** — обсуждалось, но не доведено до документа или есть открытые места
- ⬜ **Не обсуждалось** — система упоминается в лоре/контексте, но не проработана
- ❌ **Отвергнуто** — рассматривалось и решено не делать

### Условные обозначения приоритета

- 🔴 **Critical** — нужно для MVP "Погружения", блокирует разработку
- 🟠 **High** — нужно для MVP "Погружения", но не блокирует
- 🟡 **Medium** — нужно для полноценного MVP, можно начать без
- 🟢 **Low** — для "Истории" или для post-MVP
- 🔵 **Future** — на дальнюю перспективу

### Условные обозначения режима

- **[B]** — оба режима
- **[S]** — только "Погружение" (Submersion)
- **[C]** — только "История" (Campaign)

## 4. Зависимости — что от чего зависит

### Камера/управление/RT vs TB → всё остальное ✅ Зафиксировано

### Боевая система → враги → оружие → инвентарь ✅ Зафиксировано в цепочке

### Гномьи узлы → balance вылазок

Без них накопительные системы (серьёзные раны, износ оружия, заряд артефактов) не имеют точек восстановления в "Погружении". Это **критический пробел** для всего экономики ресурсов.

### LevelProfile → DataAsset'ы → PopulateCells

Без формализованного LevelProfile нельзя двигаться к технической реализации Фазы 2 генерации.

### Темница → "Погружение" MVP

Темница — стартовая точка, без неё игрок не может начать. Это блокер для playable MVP.

### Освещение → дизайн уровней + AI + стелс

Освещение завязано на множество систем. Без него стелс не работает, аномалии не подаются, атмосфера не складывается.
