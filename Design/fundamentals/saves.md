# Cursed Pit — Save System

**Статус:** Draft v0.2
**Область:** система сохранений и обработка смерти персонажа
**Применимость:** оба режима, с существенными различиями
**Зависит от:** GDD v1.0, progression_system.md, submersion_mode.md, campaign_lore.md

---

## 1. Базовый принцип

**Сохранения должны поддерживать pillar необратимости.**

Это означает:
- Никакого сейв-скама ("сохранил, попробовал, перезагрузил, если плохо")
- Смерть имеет вес
- Решения нельзя отменить простой загрузкой

При этом сохранения должны:
- Защищать игрока от технических катастроф
- Позволять прерывать игровую сессию без потерь
- Не превращаться в постоянное наказание

---

## 2. Режим "История" — permadeath с архивными копиями

### 2.1 Базовая модель

**Одно активное сохранение на персонажа.**

- Автосохранение в чекпоинтах
- Перезаписывается каждый раз
- При смерти персонажа — основное сохранение **становится завершённым**
- Новая игра начинается с пролога (или с NG+, см. п. 2.6)

**Никаких множественных слотов для одного персонажа.**

### 2.2 Чекпоинты автосохранения

Сохранение происходит в значимых точках:

- **Вход в город** — каждый раз
- **Выход из города** — каждый раз
- **Переход между уровнями** — при пересечении границы уровня
- **Перед ключевыми нарративными событиями**
- **Периодически в безопасных местах** — раз в N минут, если игрок в безопасной зоне

**В бою и в опасных ситуациях сохранение НЕ происходит.**

### 2.3 Архивные копии

Создаются в моментах **необратимых нарративных событий**:

- Пересечение точки невозврата
- Активация финальной арки
- Другие ключевые точки

**Свойства:**
- Невидимы для игрока в нормальном прохождении
- Доступны через специальное меню только если основной сейв **сломан**
- НЕ могут использоваться для отката после смерти
- НЕ могут использоваться для отмены решений

**Это НЕ механика отката после смерти.** Если игрок умер — он начинает с нуля (или с NG+).

### 2.4 Что происходит при смерти

1. Игра показывает экран смерти
2. Основной сейв помечается как завершённый
3. Игрок видит итоги (достигнутая глубина, найденные предметы, ключевые события)
4. Доступные опции:
   - **Новая игра** — с пролога, с нуля
   - **NG+** — если игрок завершал прохождение раньше (см. п. 2.6)
5. Завершённые сейвы могут отображаться в галерее (опционально)

### 2.5 Прерывание игровой сессии

- Игрок может выйти в любой момент — текущее состояние сохраняется
- При следующем входе — продолжение с того же места
- Это **не считается чекпоинтом** — это просто пауза

### 2.6 New Game+ как отдельная опция старта

После того как игрок **успешно завершил "Историю" любым финалом** (НЕ после permadeath), разблокируется опция **NG+**.

См. подробности в campaign_lore.md, раздел 10 и progression_system.md, раздел 8.

**С точки зрения системы сохранений:**
- NG+ — это **отдельный тип новой игры**
- Создаётся новый сейв-файл с пометкой "NG+"
- Стартовое состояние отличается от обычной новой игры (см. campaign_lore.md)
- В остальном правила те же: permadeath, архивные копии, одно активное сохранение
- При смерти в NG+ — снова доступна опция новой обычной игры или нового NG+

**Что НЕ делает NG+:**
- Не позволяет загрузить старого ГГ
- Не позволяет продолжить с любой точки старой истории
- Не объединяется с прогрессом старого прохождения

### 2.7 Hardcore Mode (отложено)

Возможная будущая опция (НЕ для MVP):
- Один сейв, никаких архивных копий
- Краш игры = риск потери прогресса
- Помечается в начале игры, не переключается на полпути

## 3. Режим "Погружение" — стандартный roguelike

### 3.1 Базовая модель

**Двухуровневая система:**

**Сейв текущего забега:**
- Автосохраняется во время игры
- Позволяет прервать сессию и продолжить забег
- При смерти/завершении забега — удаляется

**Мета-сейв:**
- Сохраняет всю мета-прогрессию (см. progression_system.md, п. 3.2)
- Сохраняется при изменениях
- Не теряется со смертью персонажа в забеге

### 3.2 Что входит в сейв текущего забега

- Состояние мира (генерация, состояние клеток)
- Инвентарь персонажа
- Здоровье, статусы
- Открытия в текущем забеге (карты, наблюдения)
- Текущая глубина и пройденные этажи
- **Идентификатор персонажа** (имя, предыстория — для подачи на экране смерти)

### 3.3 Что входит в мета-сейв

- Разблокированный контент (типы артефактов, SpaceTypes, Occupants)
- Бестиарий
- Типология
- Резонанс (внутренний прогресс мета-прогрессии)
- Статистика забегов (опционально)
- Архив прошлых заключённых (опционально — список имён и судеб)

### 3.4 Что происходит при смерти/завершении забега

1. Игра показывает итоги забега (имя персонажа, глубина, находки, причина смерти)
2. Обновляется мета-сейв (резонанс, новые открытия, архив)
3. Сейв забега удаляется
4. Опция: новый забег

### 3.5 Прерывание сессии

- Можно выйти в любой момент
- При возвращении — продолжение забега
- Это **не "сейв-скам"** — игрок не может перезагрузить забег, чтобы переиграть ситуацию

---

## 4. Защита от сейв-скама

### В "Истории"
- Одно активное сохранение, перезаписывается
- Архивы недоступны в нормальном прохождении
- Сохранение в чекпоинтах, а не по желанию

### В "Погружении"
- Сейв забега удаляется при смерти
- Перезагрузка приведёт ровно к тому же состоянию

### Общее
- **Никакого ручного сохранения** в обоих режимах
- **Никаких множественных слотов** для одного прохождения
- Игрок не выбирает, *куда* откатиться

---

## 5. Защита от технических катастроф

### 5.1 Резервные копии
- Перед каждой перезаписью основного сейва — **rolling backup** (1-2 последних состояния)
- Если основной сейв повреждён — игра автоматически предлагает откатиться

### 5.2 Архивные копии (только "История")
- В ключевых точках — как описано в п. 2.3

### 5.3 Защита целостности
- Сейвы пишутся атомарно (либо полностью, либо никак)
- При краше во время записи — старый сейв остаётся целым

### 5.4 Облачная синхронизация (на будущее)
- Steam Cloud при релизе

## 6. UI системы сохранений

### 6.1 Главное меню

В "Истории":
- **Продолжить** — если есть активный сейв
- **Новая игра** — всегда
- **New Game+** — только если игрок ранее завершил игру
- **Архив** — список завершённых сейвов (опционально)
- **Восстановить** — доступно только если основной сейв сломан

В "Погружении":
- **Продолжить забег** — если есть активный забег
- **Новый забег** — всегда
- **Статистика** — мета-прогрессия, бестиарий, открытия
- **Архив прошлых заключённых** — список имён и судеб (опционально)

### 6.2 Во время игры

- Никаких опций "сохранить", "загрузить" в меню паузы
- Только "продолжить", "настройки", "выйти в главное меню"
- Выход в меню сохраняет текущее состояние

---

## 7. Что цементируется этим документом

1. В "Истории" — permadeath, новая игра только с нуля или NG+.
2. Одно активное сохранение в "Истории", перезаписывается.
3. Архивные копии — страховка от технических катастроф, не механика отката.
4. NG+ разблокируется только после успешного завершения "Истории".
5. В "Погружении" — стандартный roguelike: сейв забега + мета-сейв.
6. Никакого ручного сохранения и сейв-скама.
7. Защита от технических проблем через rolling backups.

---

## 8. Что НЕ фиксируется

- Точные интервалы автосохранений
- Точный список нарративных событий, создающих архивные копии
- Точное поведение галереи завершённых сейвов / архива заключённых
- Конкретный UI/UX окон сохранений
- Стратегия облачной синхронизации
- Количество rolling backups
