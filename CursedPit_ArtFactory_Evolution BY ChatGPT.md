# Cursed Pit — Art Factory Evolution: Multi-View & 3D Pipeline

## Контекст

После анализа текущей реализации Art Factory стало понятно, что проект уже вышел за рамки обычного генератора изображений.

Сейчас система обладает рядом свойств, которых обычно нет у AI-генераторов:

- генерация основана на формализованных игровых осях;
- существуют лорные ограничения и валидация комбинаций;
- каждый результат воспроизводим через seed;
- изображения сопровождаются структурированными метаданными;
- существует контроль покрытия контента через Coverage;
- используется единый визуальный язык через IP-Adapter.

Фактически Art Factory уже является системой генерации игровых сущностей, а не просто изображений.

---

# Ключевой вывод

Следующим шагом развития НЕ должна быть немедленная интеграция Image→3D.

Наибольшую ценность даст промежуточный слой:

Axes
↓
PromptBuilder
↓
Reference Art
↓
Multi-View Generation
↓
3D Generation

То есть сначала научиться получать согласованные ракурсы объекта, и только потом строить из них 3D.

---

# Почему не прямой Image → 3D

Современные локальные решения:

- TripoSR
- Hunyuan3D
- Stable Fast 3D
- Unique3D

дают заметно лучший результат при наличии нескольких ракурсов объекта.

Для большинства генераторов:

1 изображение → 3D

работает хуже, чем:

Front / Left / Right / Back → 3D

Особенно это заметно на:

- архитектуре;
- статуях;
- механизмах;
- грибах;
- сложных пропсах.

Поэтому Multi-View способен дать больший прирост качества, чем замена самого 3D-генератора.

---

# Предлагаемый этап 5.0 — Multi-View Generator

Добавить новый режим генерации.

Вместо одной картинки Art Factory может создавать:

- Front View
- Left View
- Right View
- Back View

одного и того же объекта или пространства.

## Новый режим

- Generate Reference
- Generate Multi-View

При выборе Multi-View:

- фиксируется seed;
- фиксируются все оси;
- генерируется серия изображений;
- меняется только камера.

## Требования

Цель НЕ идеальная ортография.

Цель — согласованность формы между изображениями.

Достаточно получить:

- Front
- Left
- Right
- Back

с высокой визуальной похожестью.

---

# Новые артефакты

Помимо PNG необходимо сохранять метаданные вида:

```json
{
  "groupId": "uuid",
  "view": "front",
  "seed": 12345,
  "axes": {}
}
```

Все изображения одного набора должны иметь общий groupId.

# Этап 5.1 — 3D Export Prototype

После появления Multi-View можно добавить экспериментальный экспорт.

Пайплайн:

Multi-View Set → TripoSR → GLB → Blender Cleanup → UE5

Это НЕ должно быть частью MVP.

Это отдельный экспериментальный модуль.

- руда;
- алтари;
- факелы;
- мебель;
- контейнеры;
- статуи;

## Low Priority

# Более важное направление, чем 3D

На текущем этапе проекта большую ценность может дать не генерация моделей, а генерация анимационных референсов.

Пайплайн:

Axes → Reference Art → Video Generation → Animation Reference

Применение:

- idle;
- walk;
- attack;
- ambient behaviour.

Полученные ролики используются как референс для будущей анимации.

# Предложение: Visual Tags Layer

Сейчас sidecar содержит:

```json
{
  "axes": {},
  "derived": {},
  "seed": 123
}
```

Предлагается добавить слой визуальной семантики:

```json
{
  "visualTags": [
    "fungal_growth",
    "green_spores",
    "collapsed_bridge",
    "ancient_machinery"
  ]
}
```

## Зачем

Это позволит в будущем делать выборки не только по осям, но и по визуальным признакам.

Например:

- Показать все сцены с fungal_growth
- Сгенерировать набор ассетов для ancient_machinery

# Отдельный вывод по существам

Существ не рекомендуется пытаться встроить в текущую систему через существующие оси.

Текущие оси описывают пространство.

Существа потребуют отдельного набора параметров:

- Body Type
- Limb Count
- Eye Count
- Mutation Level
- Faction
- Threat Tier
- Depth

Поэтому для монстров и NPC следует проектировать отдельную Creature Factory, а не расширять существующий PromptBuilder.

---

# Рекомендуемый следующий шаг

Следующей крупной задачей после завершения текущего MVP считать:

**Multi-View Generator с сохранением согласованных Front / Left / Right / Back ракурсов и единым groupId.**

Ожидаемая польза:

- улучшение качества будущей 3D-генерации;
- более удобная работа художника-моделлера;
- возможность интеграции TripoSR и Hunyuan3D позже;
- фундамент для Creature Factory.
